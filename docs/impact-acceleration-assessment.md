## 3) Bottlenecks / watchpoints

### 3.1 “Write‑it‑down” bottleneck
A recurring failure mode is **unwritten constraints** (or “all lawful purposes” without a use‑restriction matrix). That invites:
- accountability gaps,
- retaliation dynamics,
- and post‑hoc rationalization.

This repo’s acceleration strategy is to keep re‑routing back to **written, reviewable** constraints and determinations.

### 3.2 Dependency‑risk vs. coercion confusion
A legitimate dependency‑risk concern can be real while still producing bad incentives if addressed through opaque pressure.

Acceleration approach: treat dependency risk as a procurement/governance problem (portability, multi‑vendor baselines, test harnesses), not as an ad‑hoc punishment system.

### 3.3 Classification as a process hazard
Classified context can swallow the record. Acceleration requires **process design** that:
- protects genuinely sensitive details,
- but still produces a reviewable core justification.
